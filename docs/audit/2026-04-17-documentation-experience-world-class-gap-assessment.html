1. Summary score

Overall DX maturity is about 8.8/10 for a PET-scale service. That is strong for static HTML docs in git.

We are not yet steady at “top few percent” (9.2+) mainly because reader feedback and design rules are still thin—not because search, IA, or governance are weak.

3. Critical gaps to close

3.1 Reader feedback is hard to see

Search telemetry exists, but pages still lack a simple control (e.g. “Was this helpful?” plus a link to file an issue).

3.2 Design is not fully standardized

Most pages use the same CSS, but section layout and visual hierarchy still vary. We need clearer page templates for steady UX.

3.3 Accessibility needs an owner for metrics

A11y checks run in CI, but we do not yet publish how often we review trends, repeat issues, or close by severity.

3.4 Onboarding path needs one owner and clear metrics

Guides exist in many places, but we still need one golden path with “you should see…” steps and a target time-to-first-success (TTFS), owned like a product artifact.

4. Design maturity review

Today we have one stylesheet, top nav, in-page TOC, and cards/tables that mostly match. To match top doc sites, the next step is enforcing patterns, not only documenting them.

Area Current state Next-level target
Page templates Mostly consistent; some drift Strict templates per type (assessment, ADR, runbook, index, how-to)
Visual hierarchy Good headings; spacing and detail vary Fixed spacing, heading rhythm, reusable blocks
Design governance Review by convention Template checks + design checklist in CI review
Feedback UX Search telemetry only Visible per-page feedback and clear issue path

5. 90-day closure plan

Phase A (0-30 days)

  • Publish search KPI policy (thresholds, warnings, what to do on breach).
  • Add per-page feedback on main hubs and assessment pages.

Phase B (31-60 days)

  • Lock doc-type templates; roll out to high-traffic pages first.
  • Publish a11y trend reports and triage SLAs by severity.

Phase C (61-90 days)

  • Ship one canonical onboarding path with TTFS checkpoints.
  • Polish design on the ~20 most-read pages.
