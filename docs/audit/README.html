Architecture & quality assessments

Overview

These are maturity assessments for this repo. Each report maps the ETR Study API to a simple list of reference practices (Table 1), scores evidence and gaps (Table 2), and lists mitigations and checklists. ADRs record decisions; these reports add a dated picture of how we compare to common practice.

Format and location

Files sit under docs/audit/ as static HTML: YYYY-MM-DD-topic-assessment.html. Naming, score bands, and the Table 2 legend are defined in ADR 0024.

Industry context

Big companies often run formal design reviews, launch checklists, security programs, and central dev portals. This repo uses a smaller, git-based checklist: Table 1 (practices), Table 2 (evidence and scores), then gaps and mitigations. It follows the same idea as large-scale maturity talks, but it does not replace automated tests, ADRs, or company-wide approvals.

Unless we say otherwise, assessments target PET-scale work. A low score often means “not done yet” or “not worth the cost now,” not “build a full platform feature.” See ADR 0024 — industry context.
