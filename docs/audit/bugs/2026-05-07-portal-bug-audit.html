Methodology and invariants

The audit was carried out as six parallel sweeps: links and anchors, OpenAPI, ADR/RFC and docs drift, JS hygiene, mobile contract and font ladder, search index and git/CI. Every finding includes an exact file:line, a category, the rationale (with a citation to the violated invariant), a reproduction path, and a fix direction (no code).

Tracked invariants

  • ADR 0020 — “PlantUML only”. Any Mermaid usage (including dormant CSS rules) counts as a violation.
  • Mobile contract (the normative #mobile-contract dated 2026-05-01): three tiers — ≥1025 / 761–1024 / ≤760; font-size declarations inside width-conditional media queries are forbidden.
  • Canonical font ladder: 8 tiers, body 0.9rem (~15.3 px), uniform across desktop / tablet / mobile.
  • Drawer ↔ sidebar. The mobile drawer must visually mirror the desktop sidebar — not degrade into a body-sized native list view.
  • RFC 0003 is the consolidated source of truth for documentation authoring.

Severity scale

  • SEV-1 — breaks core functionality (broken page, invalid OAS, memory leak, WCAG AA violation).
  • SEV-2 — UX degradation or a direct violation of a documented invariant.
  • SEV-3 — docs ↔ code drift, dead code, duplicates, minor visual bugs.
  • SEV-4 — stylistic issues, potential improvements, technical debt without present-day damage.

A [needs-verification] tag in a heading means the finding was inferred indirectly (without a full browser walkthrough) and should be verified by hand before a ticket is filed.

SEV-1 — breaks core functionality

SEV-1 · Regex lookbehind in YAML/HTTP tokenizers breaks Safari < 16.4

  • File: docs/assets/docs-syntax.js:383-384, 431
  • Category: api-compat
  • What is wrong: the lookbehind assertions (?<=:\s*) (YAML, lines 383–384) and (?<=:\s) (HTTP, line 431) are parsed at module load time. On Safari < 16.4 the literal itself is a SyntaxError, so the entire module fails to evaluate and syntax highlighting is disabled across the portal for that cohort.
  • Why it’s a bug: the file header advertises “zero-dependency, safe”, but the actual behaviour is a total breakage on Safari ≤ 16.3 (a meaningful slice of 2022–23 iOS devices).
  • How to reproduce: open any page that contains YAML / HTTP code blocks in Safari 16.3 or in an old WebKit simulator; DevTools → Console reports SyntaxError: Invalid regular expression.
  • Recommendation: rewrite the offending tokens with capture groups and post-processing, the way tokenizeBashCode() already does (line 361).

SEV-2 — UX degradation or invariant violation

SEV-2 · font-size inside width-conditional media — docs.css

  • File: docs/assets/docs.css:3976, 3980, 3990
  • Category: mobile-contract / font-ladder
  • What is wrong: three declarations live inside @media (max-width: 520px):
    • L3976 — .docs-quick-actions__list a, button { font-size: var(--fs-body); }
    • L3980 — .docs-quick-actions__item-pill { font-size: var(--fs-eyebrow); }
    • L3990 — .docs-quick-actions__item-keycap { font-size: var(--fs-eyebrow); }
  • Why it’s a bug: per mobile_contract_done.md and font_ladder_done.md, font-size is forbidden inside width-conditional media — typography must be uniform across viewports.
  • How to reproduce: viewport ≤ 520 px, the Quick Actions block.
  • Recommendation: move the rules into an unconditional selector (@layer components) or rely on cascaded inheritance from the canonical token.

SEV-2 · font-size with clamp() under media-query — home.css

  • File: docs/assets/home.css:1939, 1959
  • Category: mobile-contract / font-ladder
  • What is wrong: inside @media (max-width: 640px) the file declares font-size: clamp(2rem, 8vw, 2.8rem) and font-size: var(--fs-body). This is a triple violation: a non-canonical breakpoint, a width-conditional font-size, and a viewport-dependent clamp().
  • Why it’s a bug: the canonical 8-tier ladder (body 0.9rem) is broken on principle — clamp() returns different values at different widths.
  • How to reproduce: viewport ≤ 640 px, hero block on the landing page.
  • Recommendation: replace with a fixed tier (var(--fs-h1)) and remove L1959 to inherit from the cascade.

SEV-2 · Non-canonical breakpoints outside ≥1025 / 761–1024 / ≤760

  • Files (aggregated):
    • docs/assets/docs.css:2353 (1200 px); :3958, :5384, :5548 (520 px); :3996 (900 px)
    • docs/assets/home.css:1218 (720 px); :1641, :1929 (640 px)
    • docs/assets/internal-layout.css:175 (901–1024 sub-tier); :191, :218, :395 (900 px); :788 (600 px); :1051 (720 px); :1605 (620 px)
    • docs/assets/docs-premium.css:974 (400 px)
  • Category: mobile-contract
  • What is wrong: 14+ media queries with “foreign” thresholds fragment the three-tier contract and make visual QA non-deterministic between tiers.
  • Why it’s a bug: the normative #mobile-contract dated 2026-05-01 fixes three tiers; any deviation is SEV-2.
  • How to reproduce: 
    grep -nE '@media[^{]*(max|min)-width:\s*(?!(760|761|1024|1025)px)\d+px' docs/assets/*.css
  • Recommendation: in a single sweep, fold rules into the canonical tiers; for in-tier shifts use container queries or clamp() on properties other than font-size.

SEV-2 · Unbounded document-level listeners in docs-spec-status.js

  • File: docs/assets/docs-spec-status.js:165-166
  • Category: leak
  • What is wrong: bindOutsideClose() registers document.addEventListener("pointerdown", …) and ("keydown", …) for every pill without ever removing them. Closed <details> elements still hold their listeners; the closures retain references to the original pill.
  • Why it’s a bug: on pages with many pills (for example docs/developer/README.html), every re-render doubles the count — a real memory leak.
  • How to reproduce: reload the affected page several times, then inspect getEventListeners(document).pointerdown.length in DevTools.
  • Recommendation: use one delegated pair of listeners on the document root, or store the listener references and detach them on pill teardown.

SEV-2 · Broken anchors on resume-and-back-to-top.html

  • File: docs/internal/front/patterns/long-form-reading-aids.html:101, 106
  • Category: missing-anchor
  • What is wrong: #anatomy-banner and #anatomy-fab do not exist in the target page — only id="anatomy" is defined.
  • Why it’s a bug: the HTML fragment-id contract is violated — the link silently scrolls to the top of the page, which breaks the navigational invariant of the portal.
  • How to reproduce: click href="../components/resume-and-back-to-top.html#anatomy-fab".
  • Recommendation: either add the missing sub-section ids in the target page, or migrate the links to the existing #anatomy.

SEV-2 · OAS schemas missing description (HTTPValidationError, ValidationError)

  • File: docs/openapi/openapi-baseline.json:2517, 2527
  • Category: oas-invalid (best-practice)
  • What is wrong: two top-level schemas have no description field, which degrades introspection in Scalar / Swagger UI and downstream client generators.
  • Why it’s a bug: OAS 3.1 best practice; affects integrator DX.
  • How to reproduce: 
    jq '.components.schemas.HTTPValidationError | has("description")' \
  docs/openapi/openapi-baseline.json
# → false
  • Recommendation: add meaningful descriptions (for the 422 / RFC-7807-style validation flow), and lift a lint rule schemas-must-have-description.

SEV-2 · search-index.json drifts from real <h1> titles

  • File: docs/assets/search-index.json; affected entries: audit/2026-04-14-rest-api-assessment.html, audit/AUDIT_TEMPLATE.html, backlog/README.html, howto/README.html, internal/api/README.html, openapi/index.html, qa/0004-regression-checklist.html, qa/playbooks/api-endpoint-testing.html, rfc/0000-template.html.
  • Category: search-index / docs-drift
  • What is wrong: index titles disagree with the live <h1> on the page (different em-dash / parentheses; literal HTML entities &lt;…&gt; for the RFC template page).
  • Why it’s a bug: the search result label does not match what the user sees on the page — undermines trust in the search.
  • How to reproduce: 
    jq '.[] | select(.url=="rfc/0000-template.html") | .title' \
  docs/assets/search-index.json
    and compare with the file’s <h1>.
  • Recommendation: rebuild the index from <h1> with HTML-entity decoding; add a drift test in pre-commit.

SEV-2 · setTimeout(highlightAll, 0) as a requestIdleCallback fallback

  • File: docs/assets/docs-syntax.js:36
  • Category: race-condition / perf
  • What is wrong: on devices without requestIdleCallback (older Safari) highlighting fires through setTimeout(…, 0), blocking the next frame on large code blocks.
  • Why it’s a bug: defeats the deferred-loading strategy; produces visible jank on iOS, especially on long developer-docs pages.
  • How to reproduce: capture a Performance trace on a page with long snippets.
  • Recommendation: fall back to requestAnimationFrame, or to an idle polyfill with a 1000 ms timeout.

SEV-3 — docs ↔ code drift, dead code, duplicates

SEV-3 · Mermaid CSS rule in docs.css (dead code)

  • File: docs/assets/docs.css:1697-1701
  • Category: dead-code / mermaid-violation
  • What is wrong: the rule .sys-diagram__canvas .mermaid { … } is present even though no HTML page in docs/ uses class="mermaid".
  • Why it’s a bug: ADR 0020 — “PlantUML only” — and the feedback_diagrams_plantuml.md invariant; the rule keeps a door open to accidental Mermaid usage.
  • How to reproduce: 
    grep -n '\.mermaid' docs/assets/docs.css
grep -rn 'class="mermaid"' docs/   # → empty
  • Recommendation: delete lines 1697–1701.

SEV-3 · MutationObserver without disconnect() in home-landing.js

  • File: docs/assets/home-landing.js:607-608
  • Category: leak
  • What is wrong: themeObserver watches data-theme on documentElement and is never disconnected, even when the canvas is removed under reduced-motion or a small viewport.
  • Why it’s a bug: the proper teardown pattern is already in place in home-webgl.js (commits 5312b05, d361429); home-landing.js lags behind.
  • How to reproduce: toggle the theme 50 times and watch for FPS regression.
  • Recommendation: have bind* factories return a cleanup function; invoke it on pagehide or media-query change.

SEV-3 · Multiple addEventListener calls without removal in home-landing.js

  • File: docs/assets/home-landing.js:287-289, 602-604, 612, 756, 765, 787
  • Category: leak
  • What is wrong: skip-button click, document keydown (Escape), window resize / scroll, host pointermove / leave — all listeners hang for the page lifetime. After closeIntro() the elements are removed, but their closures keep the references alive.
  • Why it’s a bug: on long-lived sessions or SPA-style navigation the listeners pile up and may reference stale DOM.
  • How to reproduce: in DevTools observe getEventListeners(window).resize.length after several remounts.
  • Recommendation: a single bus with a teardown step; or { once: true } for one-shot listeners; for resize / scroll use a shared throttler.

SEV-3 · Empty try/catch (e) {} swallows errors in docs-nav.js

  • File: docs/assets/docs-nav.js:239, 245
  • Category: swallowed-error
  • What is wrong: localStorage.setItem(...) is wrapped in an empty catch — quota-exceeded and Safari ITP failures are silenced, so toasts do not remember their dismissal.
  • Why it’s a bug: diagnostics are impossible; the user keeps seeing toasts that should have been dismissed.
  • How to reproduce: fill localStorage to quota → dismiss a toast → reload → the toast returns and the console stays silent.
  • Recommendation: emit at least a one-shot console.warn; or annotate with /* intentional: storage may be unavailable */ plus a counter in memory.

SEV-3 · Inline blocking script without feature detection in docs/index.html [needs-verification]

  • File: docs/index.html:3
  • Category: a11y / perf (low) / swallowed-error
  • What is wrong: <script>(function(){try{… localStorage.getItem …}catch(e){}})()</script> in <head> blocks first paint, plus the empty catch. This is the well-known “no-flash theme” pattern but the catch deserves at least a comment.
  • Why it’s a bug: the pattern is justified (prevents FOUC), but without a comment it reads as a missing error path and invites false-positive corrections in future reviews.
  • How to reproduce: code review or any automated no-empty-catch rule.
  • Recommendation: add a /* no-flash theme bootstrap */ comment; do not remove the block.

SEV-3 · audit-score-legend-fragment.html is missing from the search index

  • File: docs/assets/audit-score-legend-fragment.html
  • Category: search-index
  • What is wrong: the file exists but is not in search-index.json; it does not match the existing exclusions (docs/pdoc/**, portal/people/**).
  • Why it’s a bug: either the index generator forgets fragments or there should be an explicit rule with a comment.
  • How to reproduce: 
    grep audit-score-legend-fragment docs/assets/search-index.json
# → 0
  • Recommendation: include it explicitly, or exclude it explicitly with a comment such as “component fragment, intentionally hidden from search”.

SEV-3 · HTML entities in the RFC template <h1>

  • File: docs/rfc/0000-template.html (<h1> contains &lt;number&gt;: &lt;title&gt;)
  • Category: docs-drift / search-index
  • What is wrong: the page renders literal <number> placeholders; the search index stores “RFC template” — both a drift and a meaningless heading for the user.
  • Why it’s a bug: a template should not contain a placeholder that breaks rendering; use a code fence or true placeholders inside comments.
  • How to reproduce: open the page and observe the literal text.
  • Recommendation: replace with <h1>RFC <code>NNNN</code>: <code>title</code></h1> or a plain-text template free of entities.

SEV-3 · Click handler without cleanup in docs-internal-meta.js

  • File: docs/assets/docs-internal-meta.js:176
  • Category: leak (low impact)
  • What is wrong: the “+N more” click handler is not removed before moreItem.remove().
  • Why it’s a bug: on one-off pages the impact is small, but the pattern diverges from the proper teardown in home-webgl.js.
  • How to reproduce: a profile with 4+ maintainers → expand → inspect listeners.
  • Recommendation: use { once: true } or call removeEventListener explicitly before .remove().

SEV-3 · Stray files at repo root

  • File: /PR_BODY.md, /changelog-llm-draft.md, /study_app.db, /.coverage
  • Category: repo-hygiene
  • What is wrong: the working tree is cluttered with temporary files (although .gitignore covers them). .coverage is 135 KB; study_app.db is 139 KB.
  • Why it’s a bug: it makes diff review harder and increases the chance of an accidental commit.
