UI/UX Assessment - Study App docs experience (formed 2026-04-21)

Overview

Independent UI/UX audit of the current documentation experience in ETR Study API (web docs shell, navigation, search, readability, responsiveness, trust and conversion cues). This report maps UX issues to business impact and provides a prioritized improvement plan.

Evidence source: repository review of docs structure, docs frontend behavior, and known backlog findings (including mobile navigation rendering issue on iPhone 17 Pro Max).

1. Scope and methodology

1.1 Scope

Platform assessed: documentation web experience under docs/. Core scenarios: discovery, navigation, in-page orientation, search, and content consumption on desktop/mobile.

1.2 Method

  1. Heuristic review using Nielsen heuristics, WCAG 2.1 AA, and interaction pattern standards.
  2. Flow review for high-frequency tasks: find page, scan page, jump sections, report feedback.
  3. Code-level signal check against docs frontend behavior and feature flags.
  4. Impact mapping from UX defects to CR/CTR/Task success/Time on task/retention proxies.

1.3 What was examined

  • docs/assets/docs-nav.js (navigation/search/feedback behavior and feature flags).
  • docs/internal/documentation-style-guide.html (quality bar and structure standards).
  • docs/backlog/README.html (known UX issues and priorities).
  • Current audit and index pages under docs/audit/docs/.

4. Scoring summary

4.1 Executive summary

  • Overall UX maturity: 7.2 / 10 (strong docs foundation, medium mobile/responsiveness risk).
  • Main business risk: reduced mobile engagement and higher drop-off in docs discovery path.
  • Highest-impact issue: top navigation quality on iPhone-class viewport.
  • Second risk: feedback loop is underpowered with feedback card disabled.
  • Strength: language consistency and documentation governance quality are high.

4.3 Quick wins (up to 2 weeks)

  • Ship mobile top-nav compact state and safe-area fix (expected: +10-15% mobile navigation task success).
  • Enable in-page feedback card and route to existing docs feedback workflow.
  • Add explicit empty-state guidance in search result panel.
  • Normalize touch targets for nav and quick actions to at least 44px.
