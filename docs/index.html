ETR Study API — Documentation

Overview

Welcome to the engineering documentation for the ETR Study API.

This site collects architecture overviews, engineering standards, developer onboarding guides, ADRs, runbooks, and the generated Python API documentation (using pdoc). If published (e.g., through GitHub Pages from the docs/ folder), this page serves as your documentation homepage.

⚠️ Docs changelog: Changes to the documentation are listed in docs/CHANGELOG.md. This docs changelog is distinct from the main project CHANGELOG.md, which covers product and API version changes.

Documentation index

Topic Link
Methodology Explain methodology of the project. What is ETR, why it's important, how it works, and how to use it.
Developers Docs Combined entry point: engineering workflow, quality gates, policies, and the index of hands-on guides under docs/developer/ (requirements, contracts, Make targets, local dev, Docker/K8s, docs pipeline).
Make commands and workflows Documentation on the use of make commands for setup, testing, deployments, and daily development tasks. Includes tips for new contributors and workflow automation.
Internal service documentation Engineering-facing behavior, business rules, and operational meaning for the HTTP API are documented under docs/internal/ (HTML). External integrators rely on OpenAPI and the root changelog; the split and authority rules are fixed in ADR 0025 and ADR 0026. Browse internal service documentation.
How-to guides Step-by-step walkthroughs for specific tasks, such as authentication setup, adding a new API endpoint, or navigating the internal doc layout. Geared towards practical problem-solving.
ADRs Architecture Decision Records: written rationale and trade-offs behind major technical and process-related decisions, ensuring transparency and knowledge sharing.
RFCs Deep implementation specifications and operational playbooks (for example, docs search internals: indexing, ranking, telemetry transport, persistence, troubleshooting, and KPI/SLO policy).
Architecture & quality assessments Periodic audits comparing project practices to industry standards and reference guides; includes scored tables, identified gaps, and mitigation strategies. See ADR 0024.
Runbooks Operational playbooks for handling outages, major incidents, and recovery tasks. Step-by-step guides for on-call engineers or anyone responding to operational events.
Backlog Prioritized list of features, bugs, and technical debt. Supports roadmap planning and tracking engineering work in a transparent, collaborative way.
Python API reference Auto-generated documentation for the Python codebase (via pdoc), listing modules, functions, and classes, including docstrings and usage examples.
OpenAPI (test) Interactive Swagger UI for browsing and testing the HTTP API contract. Useful for both backend/frontend developers and external integrators.
