QA checklist 0005: Accessibility (WCAG 2.1 AA)

Internal — QA portal · Checklists

Overview

Manual + tool-assisted accessibility pass for a docs page or portal feature. Walks through the four POUR principles. The procedure is in the accessibility playbook.

Preconditions Perceivable Operable Understandable Robust Pass/fail
Preconditions

Preconditions

  • Latest Chromium-based browser and one WebKit browser installed.
  • axe DevTools extension installed (or equivalent).
  • Screen reader available (VoiceOver / NVDA).
  • Page under test is loaded; reload once with cache disabled.
Perceivable

Perceivable (WCAG 1.x)

#CheckExpected
1.1Images have meaningful alt or empty alt for decorative.No image without an alt attribute.
1.2Diagrams (PlantUML) have a text caption or summary nearby.Caption present and accurate.
1.3Text contrast ≥ 4.5:1 (normal) / ≥ 3:1 (large) in light + dark.Pass for body, headings, links, code blocks.
1.4UI component contrast ≥ 3:1 (borders, focus, icons).Pass.
1.5Browser zoom 200% — no clipping, no scroll-through-scroll.Reflow correct.
1.6Information not conveyed by color alone.Status indicators use icon/text in addition to color.
Operable

Operable (WCAG 2.x)

#CheckExpected
2.1Tab through every interactive control.All reachable; order matches reading order.
2.2Buttons activate on Enter and Space; links on Enter.Pass.
2.3Escape closes drawer, lightbox, search results, popovers.Pass.
2.4Focus indicator visible everywhere; not relying on the browser default only.Pass.
2.5No keyboard trap on the page; modals trap and release on close.Pass.
2.6Skip link (if any) is the first Tab stop and visible on focus.Pass.
2.7Auto-playing or repeating motion respects prefers-reduced-motion.Animations stop or shorten.
2.8No content flashes > 3 times per second.Pass.
Understandable

Understandable (WCAG 3.x)

#CheckExpected
3.1<html lang="en"> is set.Present on every page.
3.2Page <title> identifies the page and the product.Specific and readable.
3.3Heading order does not skip levels.One h1; h2 then h3 nested correctly.
3.4Form fields have associated labels and error text where applicable.Pass (where forms exist).
3.5Predictable behavior: focus / hover does not auto-navigate.Pass.
Robust

Robust (WCAG 4.x)

#CheckExpected
4.1HTML validates (no duplicate IDs, no orphan ARIA).axe scan: no critical violations.
4.2ARIA roles only where native HTML falls short.No role="button" on a real <button>.
4.3State attributes (aria-expanded, aria-current, etc.) update on interaction.Verified per control.
4.4Landmarks present and unique-per-type or labelled.main, nav with aria-label, etc.
4.5Screen reader read-through makes sense top-to-bottom.Subjective pass with notes.
Pass/fail

Pass/fail rules

  • Pass: no critical violation; all rows pass; axe scan clean.
  • Fail: any keyboard trap, missing alt on informative image, contrast failure on body text, or missing form label.
  • Needs follow-up: minor issue with a tracked ticket (e.g. one decorative icon missing aria-hidden).
