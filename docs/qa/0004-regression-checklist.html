QA checklist 0004: Regression

Internal — QA portal · Checklists

Overview

Run this when the change set is wide (refactor, dependency bump, infra swap) or when the smoke checklist alone isn't enough confidence. This is broader than smoke and shallower than feature-level acceptance.

Scope API coverage Docs coverage Defect-prone areas Cross-cutting Pass/fail
Scope

When to run

  • Major refactor merged (touches > 1 module).
  • Dependency upgrade across the stack (FastAPI / Pydantic / SQLAlchemy / Alembic).
  • Container or runtime image change.
  • Database engine, indexing, or migration change.
  • Docs portal frontend change touching shared assets (sidebar, nav, theme).
API coverage

API coverage — golden paths for every resource

# Check Expected
1.1 User: POST → GET → PUT → PATCH → GET Each call matches its spec; final GET reflects all changes.
1.2 Conspectus: POST → GET due → POST review → PATCH Each call matches its spec; review action transitions state per spec.
1.3 Schedule summary GET Aggregates data correctly for the seeded set.
1.4 Error log: POST → GET Listing returns the just-posted error with correct shape.
1.5 OpenAPI baseline test Green; no contract drift.
Docs coverage

Docs coverage

# Check Expected
2.1 Open one page from each top-level section: ADR, RFC, How-to, Runbooks, Audit, Internal API, QA, Front docs. All render without console errors; sidebar active highlight correct.
2.2 Search index covers new pages added since last regression. Search returns each new page by title.
2.3 Theme + responsive matrix on one representative page. Light, dark, 360px, 768px, 1280px all render correctly.
2.4 Diagrams: open one PlantUML diagram in the lightbox. Renders; closes on Escape.
Defect-prone areas

Defect-prone areas — re-test these on every regression

Add to this list whenever a defect is closed. Sources: post-incident review, defect history, audit findings.

# Area Reason Re-test
3.1 Sidebar drawer focus trap Past a11y regression on Safari Open drawer, Tab through, confirm trap; Esc closes.
3.2 Search results panel close behavior Reopened twice after refactors Open results, click outside, Esc, navigate — all close cleanly.
3.3 Idempotency window edge Window math has been wrong before Replay just-after-window; treated as new request.
3.4 OpenAPI baseline drift Easy to forget when adding/changing fields Run baseline test; commit regenerated file if intentional.
3.5 Theme persistence on cross-page navigation localStorage key changes have caused reset bugs Set dark, navigate three pages, confirm dark persists.
Cross-cutting

Cross-cutting checks

# Check Expected
4.1 Error envelope on a 404, 422, 401, and one 5xx All conform per envelope.
4.2 Field conventions on three responses UUIDs, ISO-8601 UTC, snake_case keys.
4.3 Pagination on a list endpoint First page → next → empty next_cursor.
4.4 Auth boundary on three endpoints No-token / wrong-token / valid-token reach 401 / 401 / 2xx.
4.5 Migration up/down round-trip on a representative dataset Idempotent; row counts match.
Pass/fail

Pass/fail rules

  • Pass: all rows pass; new defects (if any) are S3 or below.
  • Fail: any S1/S2 found; or any defect-prone area regresses; or contract drift on the OpenAPI baseline.
  • Add to defect-prone areas any newly closed defect that surfaced through this run.
