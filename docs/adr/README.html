ADR Index

Overview

Architecture Decision Records capture what was decided, why it matters, and what was traded off — so the team does not re-litigate the same questions in chat or code review. New decisions start from the ADR template and follow ADR 0018 (lifecycle and ratification).

Process

How we record decisions

Every ADR follows a shared lifecycle: drafted, ratified via a GitHub Issue and PR, then tracked with status badges. Read these two first before opening or reviewing any ADR.

  • ADR 0018 — status model, ratification flow (Issue → PR → merge), and doc-only merge rules.
  • Template — copy-paste shell with all required sections and the status-milestone table for authors.
Documentation

Documentation & diagrams

Decisions covering where docs live, how they are generated, and how architecture is visualised.

  • ADR 0001 — docs in git with the same quality gates as application code.
  • ADR 0013 — root CHANGELOG.md, when CI requires an update, and skip tokens.
  • ADR 0016 — Google-style docstrings; pdoc generates reviewable HTML for internal modules.
  • ADR 0020 — C4 levels in PlantUML, shared skin, and Kroki-rendered PNGs in docs.
  • ADR 0024 — assessment documents under docs/audit/: format, tables, scoring, and process.
  • ADR 0025 — split between external (OpenAPI) and internal (docs/internal/) documentation.
  • ADR 0026 — docs/internal/ is the authoritative engineering narrative; history tables + git.
  • ADR 0027 — offline client-side search index: normalization model, ranking formula, and telemetry KPIs.
API & Contract

API contract & integrators

Decisions that define the public HTTP surface and the guarantees integrators can rely on.

  • ADR 0003 — stable machine-readable error codes; shared catalog aligned with OpenAPI and tests.
  • ADR 0004 — major version in the URL; additive edits inside a major; deliberate path for breaking changes.
  • ADR 0006 — critical writes use Idempotency-Key; server dedupes so retries do not double-apply business effects.
  • ADR 0007 — OpenAPI is the product-facing contract: complete examples, operation links, snapshots, and CI contract checks.
  • ADR 0022 — Superseded. Browser-side validation attempt; static explorer remains browse-only — contract checks live in CI.
Security & Config

Security, config & supply chain

Decisions covering authentication, runtime configuration, and dependency hygiene.

  • ADR 0005 — security baseline: auth strategy, rate limits, CORS allowlist, headers, and request size cap.
  • ADR 0010 — explicit dev / qa / prod profiles, layered env files, and startup validation that blocks unsafe prod settings.
  • ADR 0019 — pinned requirements.txt, pip-audit in Make/CI, severity-based response, and time-boxed exceptions.
Testing

Testing & code health

Decisions that define what must be tested, how tests are layered, and how dead code is managed.

  • ADR 0002 — API changes must include automated tests (happy path + at least one failure); unfinished work does not ship.
  • ADR 0012 — testing map: required API and contract tests in CI; load tests optional and tied to observability.
  • ADR 0014 — Ruff blocks unused imports on every PR; Vulture runs on a schedule for deeper dead-code triage.
Operations

Running, observing & SLOs

Decisions covering runtime health, metrics, structured logging, and service-level objectives.

  • ADR 0009 — /live vs /ready for orchestrators; Prometheus metrics for latency, status, and DB — same setup from laptop to prod.
  • ADR 0011 — SLIs and SLOs (availability, latency, readiness), error-budget burn signals, and Prometheus alerts.
  • ADR 0023 — NDJSON logs, X-Request-Id, optional local Elasticsearch + Kibana for search.
Workflow & Delivery

Day-to-day workflow & delivery

Decisions covering Git conventions, Make entrypoints, container images, and continuous delivery.

  • ADR 0008 — Make targets split into small primitives vs high-level fix / verify / release-check entrypoints.
  • ADR 0017 — branch naming, trunk flow on main, semver tags, and hotfix forward-porting.
  • ADR 0015 — reproducible Docker image (migrations + app entrypoint) for CI and registry pulls.
  • ADR 0021 — after green CI on main / version tags, GitHub Actions builds the image and pushes to GHCR.
