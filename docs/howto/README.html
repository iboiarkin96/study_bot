Documentation

Documentation operations

When a behavior change requires a docs update, or when you add or restructure pages under docs/internal/, use the guides below.

  1. Internal docs layout and how to add pages: Internal service docs layout.
  2. Style and audit rules: Documentation style guide and audit standard.
  3. Frontend playbooks (command palette, feedback, breakpoints, maintainers): How to change docs frontend safely.

After any docs change, run make docs-fix to regenerate artifacts, then make docs-check (or make verify) to confirm there is no drift.
