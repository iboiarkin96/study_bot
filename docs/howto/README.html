Documentation

Documentation operations

Use these guides when you change behavior and need to update the docs to match, or when you need to add or restructure pages in the internal documentation.

  1. Update internal service docs structure and navigation: Internal service docs layout and how to add pages.
  2. Validate writing style and structure against: Documentation style guide and audit standard.
  3. For operational steps when changing docs frontend (Command Palette, feedback, breakpoints, maintainers): How to change docs frontend safely (playbooks).
  4. Sync generated fragments and docs artifacts: 
    make docs-fix
  5. Run checks and drift validation before push: 
    make docs-check
