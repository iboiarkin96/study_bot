How-to guides

Practical guides for contributors: from first local run to endpoint delivery and documentation updates. This page is the canonical entry point for the Golden onboarding path and for documentation authoring workflow.

Overview

Use this index when you need a concrete, task-oriented route instead of policy documents. For architecture decisions, use ADR. For implementation specifications, use RFC. For mandatory writing and audit rules, use Documentation style guide and audit standard.

Golden onboarding path (recommended sequence)

If you are new to the repository, follow this order. It gives one complete flow from setup to review-ready PR.

  1. Run full onboarding guide: Onboarding from zero to endpoint and docs.
  2. Use the concrete endpoint implementation example: Add POST /api/v1/contract (beginner walkthrough).
  3. Update internal service docs structure and navigation: Internal service docs layout and how to add pages.
  4. Validate writing style and structure against: Documentation style guide and audit standard.

Documentation authoring path (create and format docs)

Use this checklist when you add or change endpoint behavior and must publish matching documentation.

  1. Define document type and target location (internal API page, how-to, runbook, ADR, RFC).
  2. Create or update page content using the required section structure from document-type matrix.
  3. For internal service pages, keep navigation and shell consistent with internal docs layout guide.
  4. Update links in related hubs (for example this page, internal indexes, ADR/RFC/runbook indexes) if a new page becomes part of the official flow.
  5. Run documentation build and normalization: 
    make docs-fix
  6. Run checks and drift validation before push: 
    make docs-check
