Internal service docs — layout and how to add pages

Overview

The layout of every page under docs/internal/, the assets behind the sidebar and the top nav, and the steps to add a new API resource.

Shared assets

Every page under docs/internal/ that renders the sidebar loads the three assets below and mounts #internal-sidebar-mount inside the sidebar shell.

Asset Role
docs/assets/internal-layout.css Two-column layout, sidebar chrome, and HTTP-method "tablet" styling on operation pages.
docs/assets/internal-sidebar.js The single source of truth for the left nav tree. Edit INTERNAL_SIDEBAR_NAV here when adding or renaming pages.
docs/assets/docs-nav.js The pill-style top navigation shared across every docs section.

Page anatomy

Internal pages with a sidebar set class="internal-layout" on the <body> and use a two-<div> shell. Inside .container, hub and index pages keep this order:

  1. <h1 class="internal-h1--accent-tail"> — page title with the gradient accent.
  2. <div id="docs-top-nav"></div> — top pill nav, injected by docs-nav.js.
  3. <div id="docs-page-meta-mount" hidden=""></div> — maintainer meta block.
  4. <section class="docs-page-hero" aria-labelledby="overview"> — hero with <h2 id="overview"> and an optional chip nav.
  5. One or more <section id="..." class="card"> blocks, one topic per card.
  6. <section id="page-history" class="card"> — the page-history table, last section before </main>.

The On this page outline and the report-bug FAB are injected at runtime by docs-nav.js. Do not write them into the HTML.

Canonical example: docs/internal/README.html.

How to add a new API resource

Worked example: adding Order.

  1. Create the resource folder and files:
    • docs/internal/api/order/index.html — hub page with the business context, the contract summary, and a method index.
    • docs/internal/api/order/operations/ — one HTML file per HTTP endpoint, named from the HTTP method and the path template (for example post-api-v1-order.html, get-api-v1-order--order_id.html). Do not use the Python operationId as the filename.
  2. Register in the sidebar. Open docs/assets/internal-sidebar.js and extend INTERNAL_SIDEBAR_NAV with the new paths, relative to the docs/ root.
  3. Link from the API hub. Add a card and a link on docs/internal/api/README.html.
  4. Write the content. Put business rules, field contracts, and technical notes on the resource index.html. Add one operation page per endpoint. Use internal/api/user/index.html as a reference.
  5. Run make docs-fix to normalize the HTML and rebuild the search index.

How to edit existing pages

  • Sidebar labels or order: edit INTERNAL_SIDEBAR_NAV in docs/assets/internal-sidebar.js.
  • Top navigation (Home, Internal, …): edit renderTopNav in docs/assets/docs-nav.js.
  • Page body: edit the HTML directly, then run make docs-fix to normalize the file and rebuild the search index.
  • Renaming a page: update the file path in internal-sidebar.js and docs-portal-data.js. Add a redirect stub at the old URL if outside bookmarks may exist.
