Internal service docs — layout and how to add pages

Overview

How internal pages are structured, which assets wire the sidebar and navigation, and how to add a new API resource to docs/internal/.

Shared assets

Every page under docs/internal/ that shows the sidebar must load these three assets and include the mount point #internal-sidebar-mount inside the sidebar shell.

Asset Role
docs/assets/internal-layout.css Two-column layout, sidebar chrome, and HTTP method "tablet" styling on operation pages.
docs/assets/internal-sidebar.js Single source of truth for the left nav tree. Edit INTERNAL_SIDEBAR_NAV here when adding or renaming pages.
docs/assets/docs-nav.js Top pill navigation shared across all docs sections.

Page anatomy

Internal pages with a sidebar use class="internal-layout" on <body> and a two-div shell. For hub and index pages, keep this order inside .container:

  1. <h1 class="internal-h1--accent-tail"> — page title with gradient accent.
  2. <div id="docs-top-nav"></div> — top pill nav (injected by docs-nav.js).
  3. <div id="docs-page-meta-mount" hidden=""></div> — maintainer meta block.
  4. <section class="docs-page-hero" aria-labelledby="overview"> — hero block with <h2 id="overview"> and optional chip nav.
  5. Additional <section id="..." class="card"> blocks — one topic per card.
  6. <section id="page-history" class="card"> — page history table (last section before </main>).

The On this page outline and the report-bug FAB are injected at runtime by docs-nav.js; do not author them in HTML.

Canonical example: docs/internal/README.html.

How to add a new API resource

Example: adding Order.

  1. Create the resource folder and files:
    • docs/internal/api/order/index.html — hub page with business context, contract summary, and method index.
    • docs/internal/api/order/operations/ — one HTML file per HTTP endpoint, named from method + path template (e.g. post-api-v1-order.html, get-api-v1-order--order_id.html). Do not use Python operationId as the filename.
  2. Register in the sidebar — open docs/assets/internal-sidebar.js and extend INTERNAL_SIDEBAR_NAV with the new paths (relative to docs/ root).
  3. Link from the API hub — add a card and link on docs/internal/api/README.html.
  4. Write the content — put business rules, field contracts, and technical notes on the resource index.html. Add operation pages for each endpoint. See internal/api/user/index.html as a reference.
  5. Run make docs-fix to normalize HTML and rebuild the search index.

How to edit existing pages

  • Sidebar labels or order — edit INTERNAL_SIDEBAR_NAV in docs/assets/internal-sidebar.js.
  • Top navigation (Home, Internal, …) — edit renderTopNav in docs/assets/docs-nav.js.
  • Page body — edit the HTML file directly; run make docs-fix after to normalize and rebuild the search index.
  • Renaming a page — update the file path in internal-sidebar.js and docs-portal-data.js, then add a redirect stub at the old URL if bookmarks exist.
