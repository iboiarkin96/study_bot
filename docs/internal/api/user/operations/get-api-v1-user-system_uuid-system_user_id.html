GET /api/v1/user/{system_uuid}/{system_user_id}
Status Owner Ivan Boyarkin operationId getUserBySystemUserId Last reviewed 2026-04-29 Spec template v1

Overview

Look up one user by the composite natural key (system_uuid, system_user_id). Read-only.

1. Business context

Used by integrators to verify that a learner is enrolled and to fetch the current profile (display name, timezone) before composing UI. Returns a single UserResponse.

2. User stories

  • US-USER-GET-1. As an integrator, I want to confirm an enrolment exists so I can avoid duplicate calls to POST /user.
  • US-USER-GET-2. As an integrator, I want a fresh read of the user's timezone after a profile edit.

6. Authorization & permissions

API key per shared auth. Tenant scoping is implicit via the path key being unique within system_uuid.

7. Request contract

Path parameters

system_uuid string · uuid required

External tenant UUID (RFC 4122).

system_user_id string required

External user identifier (1–128 chars; URL-encoded if needed).

Headers

X-API-Key string required

Tenant API key.

Example request

GET /api/v1/user/b2c3d4e5-0002-4000-8000-000000000002/learner-01 HTTP/1.1
X-API-Key: sk_live_***

13. Edge cases checklist

  • system_user_id with URL-special chars. Must be URL-encoded by the integrator.
  • Soft-invalidated user (is_row_invalid=1). Returned as-is; the integrator decides whether to act on the flag.
  • Idempotency-Key sent. Silently ignored.
