Error matrix

Overview

Our error management framework is built as a clear contract, using stable identifiers that guarantee both consistency and predictability. This approach allows external developers to confidently integrate with our API, knowing that error responses are reliable and version-safe even as our product evolves.

By enforcing standardized error codes, we enable precise automated monitoring, advanced analytics, and seamless troubleshooting across teams. This not only speeds up root-cause analysis and issue resolution, but also ensures that business-critical processes are not disrupted unexpectedly.

For the business, a robust and transparent error system reduces support load, strengthens compliance, and enables better SLAs for partners and internal stakeholders. Our consistent error matrix helps both engineering and non-technical teams understand system behaviors, greatly reducing time spent interpreting incidents and improving customer experience at every touchpoint.

Synchronization mechanism

  1. Define or update stable errors in app/errors/common.py and app/errors/user.py.
  2. Update OpenAPI examples in app/openapi/examples/errors.py and router responses.
  3. Run make docs-fix (calls scripts/sync_docs.py).
  4. The script patches marker blocks on this page, so catalog tables stay code-driven.
  5. Run make docs-check / make verify before merge.
