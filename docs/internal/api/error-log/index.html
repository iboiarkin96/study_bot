Error log

Overview

Single entry point for the Error log API (FR-4): product intent, ownership checks, request/response shorthand, and operation links for POST/GET /api/v1/errors.

Business context

Error log captures short learning mistakes per user and optionally links each entry to a conspectus. This supports reflection workflows and later analysis of recurring gaps.

Specification only: behavior is documented first; application code and OpenAPI for this resource are planned and will be generated after implementation.

Data contract (summary)

Field / group Role Notes
system_user_id, system_uuid Owner identity Required on both write and list operations.
message Error note payload Short learner-facing description of a mistake.
conspectus_uuid (optional) Cross-resource reference If present, must belong to the same user.

Planned implementation follows the same internal API layering pattern: middleware in app/main.py (auth/rate limit/observability), resource router, service validation, and repository persistence. Write operations require idempotency; list operation is read-only.
