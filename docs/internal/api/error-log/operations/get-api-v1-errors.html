GET /api/v1/errors

Error log

GET /api/v1/errors
Status Owner Ivan Boyarkin operationId listErrorLogs Last reviewed 2026-04-29 Spec template v1

Overview

List the learner's learning errors, optionally filtered by conspectus_uuid. Read-only; bare array, ≤ 100 items per call (per shared pagination).

1. Business context

Powers the “open loops” surface in the client UI: the queue of weak cues / mistakes the learner promised to address. Filtering by conspectus_uuid drives the in-note error sidebar so the next Transform pass is targeted.

2. User stories

  • US-ERR-LIST-1. As a learner, I want to see all outstanding errors so I can plan the next Transform pass.
  • US-ERR-LIST-2. As a learner, viewing a conspectus I want to see only errors tied to that note.

3. Acceptance criteria (Given / When / Then)

ID & scenario Given When Then
AC-ERR-LIST-001
All errors. 		U has 4 errors (mixed standalone and linked). GET without filters. 200 OK with 4 items, ordered created_at DESC, error_uuid DESC.
AC-ERR-LIST-002
Filter by conspectus. 		U has 4 errors, 2 tied to conspectus C. GET ?conspectus_uuid=C. 200 OK with the 2 linked items.
AC-ERR-LIST-003
Filter by another user's conspectus. 		Conspectus C owned by U1; request resolves to U2. GET as U2 with ?conspectus_uuid=C. 200 OK with [] (no leak; treat as “learner has no errors for that ID”).
AC-ERR-LIST-004
User not found. 		No user matches the query composite. GET. 404 USER_404.
AC-ERR-LIST-005
Cap at 100. 		U has 250 errors. GET. Exactly 100 items (newest 100).

4. Functional requirements

ID Short description Detailed characteristic
FR-ERR-LIST-001 Resolve owner from query. Composite key resolution; 404 USER_404 when not found.
FR-ERR-LIST-002 Implicit owner scope. WHERE owner_client_uuid = ?; cross-tenant rows are not visible.
FR-ERR-LIST-003 Optional conspectus_uuid filter. AND conspectus_uuid = ?; if the conspectus belongs to another tenant the query naturally returns zero rows (no leak).
FR-ERR-LIST-004 Deterministic ordering. ORDER BY created_at DESC, error_uuid DESC.
FR-ERR-LIST-005 Cap at 100. Hard LIMIT 100.
FR-ERR-LIST-006 Bare array response. JSON array of ErrorLogResponse; empty array allowed.

5. Non-functional requirements

ID Short description Detailed characteristic
NFR-ERR-LIST-001 Latency. P95 ≤ 150 ms.
NFR-ERR-LIST-002 Index usage. Composite (owner_client_uuid, created_at DESC); optional secondary (conspectus_uuid, created_at DESC).
NFR-ERR-LIST-003 Cap. Hard LIMIT 100.
NFR-ERR-LIST-004 Read consistency. Default DB isolation.
NFR-ERR-LIST-005 Backwards compatibility. Adding query params is additive.
NFR-ERR-LIST-006 Observability. Standard request_done.
NFR-ERR-LIST-007 Rate limit. Inherits global limit.
NFR-ERR-LIST-008 Cacheability. No caching in v1.

6. Authorization & permissions

API key per shared auth; learner from query. Result implicitly scoped to owner_client_uuid.

7. Request contract

Headers

X-API-Key string required

Tenant API key.

Query parameters

system_user_id string required

External user identifier.

system_uuid string · uuid required

External tenant UUID.

conspectus_uuid string · uuid optional

Restrict to errors linked to this conspectus.

Example request

GET /api/v1/errors?system_user_id=learner-01&system_uuid=b2c3d4e5-0002-4000-8000-000000000002 HTTP/1.1
X-API-Key: sk_live_***

13. Edge cases checklist

  • Filter by conspectus that belongs to another tenant. 200 [] (no leak).
  • 250 errors. Cap at 100; integrator polls again with created_at filter (added later).
  • Conspectus deleted (FK SET NULL). Errors still appear in the global list with conspectus_uuid=null.
  • Idempotency-Key sent. Silently ignored.
