POST /api/v1/errors

Error log

Summary Records a single learning error for later remediation. Status Availability v1 · Internal Owner Ivan Boyarkin operationId createErrorLog Last reviewed 2026-04-29 Spec template v1

Overview

Record one learning error — a weak cue or mistake the learner wants to remediate later (per methodology · error log). Optional conspectus_uuid ties the entry to a specific note when both are owned by the same learner.

1. Business context

The error log is the bridge between Retrieve outcomes and the next Transform pass: instead of just rating a review hard/forgot, the learner captures why in their own words. The next time they edit the conspectus, the error log seeds what to fix (methodology mapping; decision D5).

2. User stories

  • US-ERR-CREATE-1. As a learner, after a failed retrieve I want to write a short note about the gap so my next Transform pass targets it.
  • US-ERR-CREATE-2. As an integrator, I want creation to be idempotent so a network retry never duplicates a learner's mistake note.

6. Authorization & permissions

API key per shared auth; learner from body. Ownership rules: referenced conspectus_uuid / review_log_id must be owned by the resolved learner; cross-tenant references collapse to 404 ERR_404.

10. Side effects

  • On success. One transaction: INSERT INTO learning_errors; INSERT INTO idempotency_keys; emit error_log_created (info).
  • On idempotent replay. No DB writes; replay stored 201; emit error_log_create_idempotent_replay (info).
  • On error. Rollback; no partial state.

11. Idempotency & concurrency

  • Namespace: POST /api/v1/errors.
  • Key composition: Idempotency-Key + SHA-256 of canonical body.
  • Replay status: 201 with stored body.
  • TTL: 24 h.
  • Concurrency: not applicable — fresh row, server-generated UUID.

13. Edge cases checklist

  • Standalone (no FK). Accept; both conspectus_uuid and review_log_id are null.
  • Referenced conspectus is later deleted. FK is SET NULL; the error row survives.
  • conspectus_uuid set, review_log_id not owned. 404 ERR_404 from the second check.
  • Long unicode message. Accept up to 2000 chars (UTF-8 bytes).
  • Message with only whitespace. Reject after trim (422).
  • Idempotency replay across conspectus deletion. Stored 201 still served; current FK state in DB may differ from the replayed body.
