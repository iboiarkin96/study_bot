Overview

This page is a fast navigation map for contributors who need to quickly understand which frontend documentation to read first based on their role and task type.

For canonical term definitions, see Docs frontend glossary.

Path for analysts and product owners

  1. Start with architecture map and feature inventory for full feature coverage and ownership zones.
  2. Read normative navigation and theme contract to understand behavior-level product commitments.
  3. Review navigation/search/discovery when planning IA or findability changes.
  4. Review feedback and editorial workflow for intake and telemetry expectations.

Path for frontend contributors

  1. Start with runtime ownership map to identify target JS/CSS files.
  2. Open UI kit and UI/motion/adaptivity before editing visual behavior.
  3. Use navigation/search/discovery and hotkeys for interaction changes.
  4. For diagram UX work, use diagrams and lightbox.
  5. Before merge, follow safe change playbooks.

Path for reviewers and maintainers

  1. Check impacted DFEAT-xxx IDs in feature ledger.
  2. Verify expected behavior against contracts: normative contract, telemetry contract, and page-specific sections.
  3. Use playbooks for acceptance checks and cross-file update requirements.
  4. Ensure ownership metadata remains correct via Edited by workflow.
