Overview

This page is the entry point for frontend documentation of the docs site. It maps all currently implemented frontend capabilities, where they are implemented, and which specialized pages describe each block in detail.

Scope is the static documentation frontend under docs/: HTML mounts, shared JavaScript runtime, CSS design system, navigation/search, UX helpers, and editorial workflow widgets.

Complete frontend feature inventory (implemented)

  • Top navigation with two groups (Project, Code), active-link state, breadcrumbs, and docs search.
  • Internal sidebar tree with nested API resources, group expansion logic, desktop collapse mode, and mobile/tablet drawer mode.
  • Menu launcher in title row for internal-layout pages, with ARIA synchronization to drawer open/close state.
  • Theme cycle: automatic -> light -> dark, flashlight icon control, dark-mode lit state, and toast status feedback.
  • Desktop-only page actions toolbar and command palette with keyboard-first navigation and grouping by action type.
  • Global docs search (client-side inverted index), ranking boosts, did-you-mean suggestions, and telemetry events.
  • Reading mode toggle and reading progress bar for long pages.
  • Auto in-page TOC generation from headings, anchor normalization, and active-section scroll spy.
  • Rocket back-to-top control that appears near bottom and launches upward on click.
  • Continue reading prompt (resume previous position), persisted in local storage with expiration policy.
  • Page feedback card with type selector, validation, clipboard fallback, and prefilled GitHub issue creation.
  • Edited by widget with avatar stack and expandable editor list resolved from generated people data.
  • Portal widgets for people grouping and profile maintained-pages pagination with skeleton states.
  • Diagram lightbox for PlantUML images with expand button, full-screen dialog, zoom controls, and drag-to-pan.
  • Skip-to-content accessibility link and reduced-motion behavior for animated components.

Feature ID governance

  • Use DFEAT-xxx IDs when discussing frontend changes in tasks, PRs, and review comments.
  • IDs are stable and must not be renumbered after publication; append new IDs at the end.
  • If a feature is superseded, keep its ID and mark the row as superseded instead of deleting it.
  • When adding a new feature, update this ledger and the relevant detailed page in the same change set.
