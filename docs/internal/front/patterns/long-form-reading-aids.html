Pattern: long-form reading aids Pattern

Docs portal · Patterns

Overview

Long internal pages — system-design, deep ADRs, big API specs — routinely run past 5 000 words. The reader's three pain points are: where am I in the page, how do I get back to the top, and how do I resume reading after a tab switch or refresh. This pattern composes three existing UI parts into one coherent reading experience that addresses all three.

When to use this pattern

  • Apply on any internal page whose content runs longer than two viewport heights at desktop width (roughly 1 600 px of content).
  • Apply on pages that act as references or specifications — readers leave and return repeatedly.
  • Skip on hubs, indexes, and short summary pages where the reader is unlikely to scroll far.
  • Skip on pages where a sticky in-page TOC already handles place-keeping (the TOC is a stronger affordance and these aids would compete with it visually).

Components participating

The pattern composes three building blocks already documented in their own component pages. Each component is the source of truth for its own anatomy and accessibility; this page only describes how they work together.

Component Role in this pattern Trigger
Resume banner Restores the reader's previous scroll position on first load. DOMContentLoaded + saved position within the 14-day TTL.
Back-to-top FAB One-click return to the top after the reader has scrolled. scrollY > 600 px.
Rocket launch animation Transient state of the FAB on activation: signals the page is scrolling back up. FAB click while armed.

Reader's journey

  1. First load. If a saved scroll position exists for this URL within the 14-day TTL, the Resume banner appears at the top of .internal-layout__main. The reader either clicks Resume reading at section X (a smooth scroll moves the page to the saved offset) or dismisses the banner.
  2. Reading. No aids interfere with reading. The runtime quietly tracks the deepest scroll position and persists it on tab close / unload.
  3. Past the threshold. When scrollY > 600 px, the Back-to-top FAB fades in at the bottom-right of the viewport (above the report-bug FAB).
  4. Return to top. On click, the FAB switches to its Rocket launch state — a short transient ornament — while the page smooth-scrolls to 0,0. Animation respects prefers-reduced-motion: reduce.
  5. Leave and return. If the reader closes the tab and returns within 14 days, the journey starts again at step 1 with the resume banner offering the saved offset.

Acceptance checklist

  1. Banner shows once. Returning to a page within the TTL shows the banner; clicking Resume or Dismiss hides it for that visit.
  2. Banner does not show twice. Once dismissed, the banner stays hidden until the reader loads the page in a fresh visit.
  3. FAB threshold. The back-to-top FAB is hidden under 600 px of scroll and visible at or above it. No flicker at the boundary.
  4. FAB does not overlap content. Bottom-right placement keeps clear of the report-bug FAB and any sticky footer; safe-area-insets are honoured on mobile.
  5. Rocket transient is brief. The launch animation completes in under 1 200 ms and releases focus back to the document afterwards.
  6. Reduced motion. With prefers-reduced-motion: reduce, the rocket animation is replaced by an instant scroll-to-top; the resume banner uses no animation on entry.
  7. Drawer overlay precedence. When the mobile drawer is open (z-index 1000), it covers both the banner area and the FAB.
  8. Keyboard. The banner's primary action and dismiss button are reachable by Tab; the FAB activates on Enter and Space; Escape after activation does not strand focus.

Implementation entry-points

  • docs/assets/docs-resume.js — banner runtime, scroll-position persistence, FAB visibility threshold.
  • docs/assets/docs-nav.js — finishLaunch() closure for the rocket transient, FAB armed/launching state machine.
  • docs/assets/docs.css — selectors prefixed .docs-resume-banner*, .docs-back-to-top*, and the launch ornament classes.
  • docs/assets/internal-layout.css — layout slot for the banner inside .internal-layout__main; safe-area-inset rules for the FAB cluster.

Opt-in per page: the runtime mounts only when a page declares it; opting in is a single attribute on <body>. See the component spec for the exact declaration.

Anti-patterns

  • Don't pair this pattern with a sticky in-page TOC. The TOC already keeps place visually and links every section. Showing both adds clutter without solving an unmet need.
  • Don't widen the resume banner with a long primary action. The banner is meant to be dismissable in one glance; multi-line actions push real content below the fold.
  • Don't trigger the rocket transient on every scroll-up. The animation is reserved for the explicit FAB click — using it as a passive scroll signal devalues the affordance.
