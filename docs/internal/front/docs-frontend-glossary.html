Overview

This glossary defines frontend documentation terms used across the docs platform so contributors, analysts, and reviewers share the same language when discussing behavior and changes. Every cheatsheet section pairs each term with a plain-language definition, an implementation hook, and a small inline-SVG visual so concepts like hover, focus ring, WebGL, scramble, parallax, magnetic CTA, color-mix, focus trap, or IntersectionObserver are unambiguous on first read.

Visual cheatsheets, in order of how a contributor encounters them:

  • Interaction states — hover, focus, active, disabled, loading, drag, selected, hit area.
  • Rendering primitives — WebGL, canvas, rAF, easing, clip-path, stacking context, backdrop-filter, mask, compositing layer, prefers-reduced-motion.
  • Layout primitives — flexbox, grid, position fixed/sticky/absolute, aspect-ratio, overflow, container query.
  • Typography & text effects — variable font axis, tracking, tabular-nums, ellipsis, caret, dotted underline.
  • Color & gradients — linear/radial/conic, color-mix, theme tokens, contrast pair.
  • Motion patterns — slide, fade, scale, stagger, parallax, shimmer, pulse, scramble, magnetic, iris shutter, lock-flash, scan beam, scanlines + vignette.
  • Accessibility patterns — skip link, focus trap, sr-only, ARIA live region, landmark, aria-current, roving tabindex.
  • Performance & observers — IntersectionObserver, ResizeObserver, MutationObserver, throttle, debounce, layout shift, reflow.
  • Forms & input patterns — text input, select, combobox, listbox, checkbox, radio, toggle, accordion, tabs, dropdown, popover, modal, drawer, sheet, FAB.
  • Iconography — chevron, caret, arrow, plus, close, hamburger, kebab, search, info, warning, success, danger.
  • Status & badges — chip, success/warning/danger/muted badge, notification dot, page-type badge.
  • Theme tokens gallery — --bg, --card, --text, --muted, --accent, --line, --focus-ring, --docs-overlay-*, badge tokens, motion tokens (light + dark swatches).
  • Docs portal primitives — toast stack, sticky TOC, breakpoint, skeleton.

Home screen terms (engineering hub landing)

Terms specific to the docs portal home screen (docs/index.html) and its boot intro overlay. Each term has a short, plain-language definition first, then a precise implementation note. If you don't know a term while reading code or specs, look it up here before guessing.

Term Plain-language definition Implementation note Reference
Hero The large, attention-grabbing block at the top of a landing page. Holds the title, tagline, and primary CTAs. .home-hero in home.css; isolated stacking context with background layers behind and content grid in front. Home spec — Hero
Eyebrow A short uppercase label sitting above a heading, used to label the section ("Engineering Documentation"). .home-hero__eyebrow; small, accent-colored, with a leading 18×2px accent bar. Home spec — Eyebrow
Tagline One-line subtitle below the H1 that summarizes the page's purpose. .home-hero__tagline; muted color, max-width 58 characters. Home spec — Tagline
Wordmark A textual logo: the product name set in a distinctive type style, treated as a brand mark. The home boot intro renders ETR // API as a wordmark with per-character spans for the scramble effect. Home spec — Wordmark scramble
Glyph A single visual character — a letter, digit, symbol, or pictogram. Used in this codebase to mean "one drawn character" (e.g. one cell of the rain canvas, one slot of the wordmark). —
Glyph rain (Matrix rain) A background of falling random characters in vertical streams, styled after the "code rain" from The Matrix. Canvas-based; columns of glyphs (Katakana + ASCII) fall at random speeds with bright head + faded trail. Implemented in startGlyphRain(). Home spec — Glyph rain
Glyph scramble (decrypt effect) A reveal animation where a piece of text first cycles through random characters and then "settles" into the real text, character by character. Used in two places: the intro wordmark (startWordmarkScramble) and the H1 (bindHeroDecrypt). Each char rolls random glyphs every ~50ms, then locks at a per-char timestamp. Home spec — Wordmark scramble
Lock-in / lock-flash The moment when a scrambling character settles to its final letter, accompanied by a brief brightness flash and scale pulse. CSS animation ig-lockflash (scale 1.18 → 0.98 → 1, 380ms). Triggered by adding .ig-char--locked. Home spec — Wordmark scramble
HUD bracket An L-shaped corner mark used in heads-up displays (HUD) to frame an object of interest, like the targeting reticle in a video game. Four .home-intro__bracket elements at corners of .home-intro__stage; idle pulse + one-shot snap on full lock. Home spec — HUD brackets
Crosshair tick A short vertical line above and below an object, marking its center axis — like the tick marks of a sniper scope. .home-intro__crosshair--top/bot at the vertical center of the HUD stage. Home spec — HUD brackets
Radar pulse ring (sonar ring) A circle that grows from a center point and fades out, mimicking a radar or sonar ping. Three .home-intro__radar elements expand from scale(0.5) to scale(7.2) over 3.6s, staggered 0/1.2/2.4s. Home spec — Radar rings
Vignette A dark border-fade around the edges of an image or screen that pulls the eye toward the center. .home-intro__vignette — radial gradient that darkens the outer 35% of the overlay. Home spec — Boot intro
Scanlines Faint horizontal stripes overlaid on a screen, evoking a CRT television or technical instrument display. .home-intro__scanlines — 1px-on / 3px-off repeating linear gradient at low opacity. Home spec — Boot intro
Iris shutter A circular wipe that closes (or opens) like a camera aperture, used here to dismiss the intro overlay. runShutterExit() animates a clip-path: circle(150% → 0%) over 720ms with cubic-bezier(0.87, 0, 0.13, 1). Home spec — Iris shutter
Mask reveal An animation that reveals content by uncovering it from behind a moving mask — a wipe, a curtain, or a clip-path. Pattern used in earlier intro iterations; current intro uses iris shutter on exit and per-character scramble on entry. —
Skip button (skip control) A button that lets the user dismiss an intro/animation early instead of waiting it out. .home-intro__skip; absolute top-right; Esc bound as keyboard equivalent. Home spec — Skip
Status pill A small rounded label showing the current state of a system. Often paired with a colored dot. .home-intro__status — monospace text with a pulsing cyan dot; transitions to steady white on .is-locked. Home spec — Status pill
Progress bar A horizontal bar that fills from 0% to 100% to show how much of a process is complete. Two on this page: (a) .home-intro__bar tied to wordmark glyph lock count; (b) .home-scroll-progress tied to scroll position via scaleX. Home spec — Progress bar
Scroll progress bar A thin bar fixed at the top of the viewport that fills as the user scrolls down the page. .home-scroll-progress; updated each scroll/resize via rAF in bindScrollProgress(). Home spec — Scroll progress
CTA (Call To Action) A clickable button or link inviting the user to take a primary action ("Start", "Sign up", "Open the docs"). .home-hero__cta; primary uses gradient fill + magnetic effect, secondaries use card border. Home spec — CTAs
Magnetic CTA A button that subtly slides toward the cursor when the cursor is near, as if attracted to it. Opt-in via data-home-magnetic; pointer offset is mapped to a damped translate, clamped to ±9/±7px. See bindMagneticCta(). Home spec — Magnetic CTA
Mono-chip A small rounded pill of monospace text used to display short metadata (a version, an environment, a tag). .home-chip; backdrop-blurred 6px, monospace 0.7rem, optional leading status dot. Home spec — Mono-chips
Number ticker (count-up) A number that animates from zero up to a target value when it scrolls into view. .home-ticker__num with data-home-ticker-target + optional data-home-ticker-suffix; cubic-out easing over 900ms; stagger 80ms; uses tabular-nums. Home spec — Tickers
Terminal panel (live code panel) A card styled like a terminal window (with traffic-light dots and monospace text) that shows a code or shell example. .home-hero__terminal; macOS chrome, syntax-colored typing of TERMINAL_SCRIPT via bindLiveTerminal(); gated on intro-done + visibility. Home spec — Terminal
Caret (typing cursor) The blinking vertical bar shown at the current insertion point in a text editor or terminal. .home-terminal__caret 8×1em block; blinks via @keyframes cursor-blink at 720ms steps(2, end). —
Cursor-reactive dot grid A regular grid of small dots that brightens and grows where the cursor is hovering, as if the cursor lights up nearby dots. Canvas-based; spacing 26px, react radius 140px. Implemented in bindDotGrid(); rAF-throttled, theme-aware. Home spec — Hero background
Parallax orb A blurred glowing disc placed in the background that drifts slightly when the user moves the cursor, creating a sense of depth. .home-hero__orb--one/two/three; depth multipliers 0.28 / 0.18 / 0.12. Driven by bindHeroParallax() with damping 0.12. Home spec — Hero background
Section tint shift A subtle change in the background color tone between scroll sections, used to give each section a distinct mood. [data-section-tint="cool|teal|mono"]; .is-tinted applied via IntersectionObserver; tint rendered in ::before as a radial gradient. Home spec — Section tints
Card stagger reveal A reveal animation where cards in a row fade/slide in one after another with a small time delay, instead of all at once. bindRevealOnScroll() sets transition-delay: i * 65ms per card and adds .is-visible in a double-rAF. Home spec — Card sections
Shimmer sweep A diagonal translucent band that sweeps across a card on hover, like light catching the surface. .home-card::after; translateX(-110% → 110%) over 620ms. Home spec — Card hover
3D card tilt A small 3D rotation applied to a card on hover so it feels like it's leaning toward the cursor. .home-card:hover applies translateY(-3px) rotateX(1.2deg) rotateY(-1.4deg) (motion-allowed only). —
Variable font (wght axis) A font file that can interpolate between many weights/widths along an axis, instead of shipping discrete weight files. font-variation-settings: "wght" var(--home-h1-wght) on .home-hero__title; --home-h1-wght animated 500 → 760 on scroll. Home spec — H1
View Transitions API A browser API that animates the change between two states — including the transition between two pages — automatically. Currently Chromium-only. document.startViewTransition() in bindViewTransitions(); per-CTA view-transition-name via CSS for shared-element morphs. Home spec — CTAs
Scan beam A bright bar that sweeps once across an element, like a barcode scanner reading the wordmark. .home-intro__wordmark-beam; one-shot animation ig-beam on .home-intro__wordmark.is-locked. Home spec — Wordmark
Reduced motion A user preference (operating system setting) that asks websites to minimize non-essential animations. Respecting it is an accessibility requirement. @media (prefers-reduced-motion: reduce) gate; everything decorative is disabled or simplified, intro is removed, terminal renders statically. Home spec — Accessibility
Anatomy diagram A labelled illustration that names every visible region of a screen or component, like a wiring diagram for the UI. Annotated SVG embedded in screen specs (preferred over screenshots — version-controllable, doesn't go stale on style tweaks). Home spec — Anatomy
Screen spec A single-page document that fully describes one screen of the product: layout, behavior, data hooks, edge cases, and where to change it. Lives in docs/internal/front/screens/docs-screen-*.html. Always start from the template. Screen spec template

Interaction states (visual cheatsheet)

These are the states every interactive surface in the docs portal can be in. Each row pairs the plain-language definition with the implementation hook and a small inline-SVG mockup so the visual is unambiguous. Use these terms verbatim in PR descriptions, screen specs, and bug reports.

Term Plain-language definition Implementation note Visual example
Default (rest) The neutral, untouched look of a control. No pointer near it, no keyboard focus, not pressed. The base CSS rule for the selector — for example .btn { … } with no pseudo-class.
Hover The cursor is over the element but no button is held down. A reversible visual hint that the element is interactive. :hover pseudo-class. Use only for purely cosmetic feedback — do not put logic behind it; touch and keyboard users never trigger :hover.
Focus The element is the keyboard's current target — the next Enter or Space will activate it. :focus pseudo-class. Required for every interactive element; never outline: none without a replacement focus indicator.
Focus-visible The element is focused and the user got there by keyboard or assistive tech. Mouse-click focus is suppressed so a sighted mouse user does not see a ring after every click. :focus-visible pseudo-class. Preferred over :focus for the visible ring; pair with :focus for AT compatibility.
Active (pressed) The user is currently pressing the element — pointer is down, finger is on it, or Space/Enter is held. :active pseudo-class. Usually a short press-down look (slight scale 0.98 or darker fill); released as soon as the input is released.
Disabled The control exists but is intentionally inactive — no hover, no focus, no clicks. disabled attribute on form controls; aria-disabled="true" for ARIA equivalents. Greyscale + reduced opacity; pointer-events disabled is implicit on native controls.
Loading / busy The control is waiting on an async operation. Visible feedback so the user does not retry or assume the click was ignored. aria-busy="true" on the region; usually a spinner replaces the label or a skeleton bar replaces the content. Disable secondary clicks while busy.
Drag (in-flight) The pointer is held down on an element and moving — the element follows the cursor as a translucent ghost. The original position usually shows a drop-zone indicator. HTML drag-and-drop API or pointer-events JS. Mark the row with aria-grabbed + a data-dragging hook so drop targets can react.
Selected / current The element is the active item in a list, tab strip, or TOC. Persistent across interactions — unlike :active which lasts only while pressed. Project convention: the --active modifier class (for example .docs-inpage-toc__link--active) plus aria-current="true".
Hit area / target size The clickable region around a control — usually larger than the visible glyph so touch users hit it reliably (≥44×44 CSS px is the WCAG target). Use padding or transparent ::before overlays to expand the click region without changing the visual size.

Rendering primitives (visual cheatsheet)

The docs portal includes WebGL backgrounds, canvas grids, and bespoke CSS effects. These rows give a plain-language definition for the underlying browser primitive plus a tiny SVG that hints at what the effect looks like, so a contributor can read code like requestAnimationFrame or backdrop-filter and form the right mental model immediately.

Term Plain-language definition Implementation note Visual example
WebGL A browser API that lets JavaScript draw 2D or 3D graphics on the GPU using shaders. Used for the home-screen background and any pixel-rate animation that would melt a CPU. Entry point is canvas.getContext("webgl2"); shaders are written in GLSL strings. See docs/assets/home-webgl.js.
Canvas (2D) A browser-drawn pixel buffer. JS draws shapes, text, and gradients into it imperatively. Faster than DOM for thousands of small elements (rain, grid, sparks). canvas.getContext("2d"); redraws every frame inside requestAnimationFrame. See startGlyphRain().
requestAnimationFrame (rAF) A browser scheduling hook that calls your function once before the next paint — usually every ~16 ms (60 Hz). Use it for any visual update; never use setInterval for animation. requestAnimationFrame(callback) returns a handle you can cancelAnimationFrame. Throttle pointer-driven UI inside rAF to coalesce multiple events into one paint.
Easing curve (cubic-bezier) A function that maps animation progress (0 → 1) to a non-linear curve, so motion feels physical instead of robotic — fast start, slow stop, etc. CSS transition-timing-function: cubic-bezier(.4, 0, .2, 1); project tokens in --motion-ease-* (see docs.css).
Clip path A shape that defines which parts of an element are visible. Anything outside the shape is hidden — no actual pixels are removed, just masked. CSS clip-path: circle(150% at center); the home iris-shutter exit animates the radius to 0%.
Stacking context (z-index) A 3D layering of elements. Elements in higher stacking contexts paint on top of lower ones, regardless of source order. Misunderstanding this is the #1 reason a popup ends up behind a sticky bar. Created by position + z-index, by opacity < 1, by transform, by filter, and by other CSS properties. Project z-index ladder is documented in CSS architecture.
Backdrop filter A blur or saturate effect applied to whatever sits behind a translucent surface. Lets a card appear frosted glass over the page. CSS backdrop-filter: blur(12px) saturate(1.08); used on toasts and popups to keep them legible over busy backgrounds. Safari needs the -webkit- prefix.
Mask (gradient mask) Alpha-fades part of an element. Used for fade-to-edge text, shimmer sweeps, and edge softening on long lists. CSS mask-image / -webkit-mask-image with a linear or radial gradient. Black = fully visible, transparent = fully hidden.
Compositing layer (will-change) A hint to the browser that an element will be animated and should be promoted to its own GPU layer to avoid repainting siblings every frame. will-change: transform, opacity. Use sparingly — every layer costs memory; over-promoting can hurt performance.
prefers-reduced-motion An OS setting users enable when motion makes them dizzy or distracted. Browsers expose it; respecting it is non-negotiable. CSS @media (prefers-reduced-motion: reduce). Replace transforms with instant state changes; never just slow them down.

Layout primitives (visual cheatsheet)

Core CSS layout idioms used across the docs portal. The visuals show the resulting box arrangement, not the property syntax.

TermPlain-language definitionImplementation noteVisual example
Flexbox (row) A 1D layout that places children in a row (or column), distributing space along the main axis. The bread-and-butter layout for toolbars, action rows, chips. display: flex; gap: 8px; justify-content: …; align-items: …
CSS Grid A 2D layout that places children into rows and columns of an explicit track grid. Used for page-level scaffolding and card grids. display: grid; grid-template-columns: repeat(3, 1fr); gap: 12px
Position fixed Pinned to the viewport — does not scroll with the page. Used for the toast stack, the rocket back-to-top, the scroll progress bar. position: fixed; right: 22px; bottom: 22px;
Position sticky Scrolls with the page until it hits a threshold, then stays in place — like a header that "freezes" on scroll. Used for the in-page TOC and the page top bar. position: sticky; top: 0; — needs a scrollable ancestor.
Position absolute Pinned to the nearest positioned ancestor (not the viewport). Used for badge overlays, dropdowns, decorative absolutely-placed accents. position: relative on the parent + position: absolute; top:0; right:0; on the child.
Aspect ratio A box whose height is derived from its width (or vice versa) so it keeps a fixed shape across breakpoints — used for diagram thumbnails and SVG figures. aspect-ratio: 16 / 9; on the container.
Overflow (clip / scroll) Decides what to do when a child does not fit: hide it, scroll it, or let it spill. .docs-table-scroll wraps wide tables in a scroll container so they do not break the layout. overflow: hidden clips; overflow: auto shows a scrollbar; overflow: visible spills.
Container query A media query that reacts to the size of the parent, not the viewport. Lets a card adapt when it's used in both a wide and a narrow column. @container (min-width: 480px) { … } with container-type: inline-size on the parent.

Typography & text effects (visual cheatsheet)

Type-level idioms — the ones that show up in our CSS and content rules.

TermPlain-language definitionImplementation noteVisual example
Variable font axis (wght) One font file that interpolates smoothly between weights — 300 to 900 — without shipping eight different files. font-variation-settings: "wght" 600; animatable as a custom property.
Tracking (letter-spacing) The extra horizontal space between every letter. Tighter tracking feels modern and large; looser tracking feels formal and small. letter-spacing: 0.01em; on titles; 0.18em uppercase for eyebrows.
Tabular nums Digits that all take the same width so columns of numbers (counters, tickers, table cells) line up vertically. font-variant-numeric: tabular-nums; on counters and tables.
Ellipsis truncation Cuts a long string and ends it with "…" so the row keeps a fixed width. white-space: nowrap; overflow: hidden; text-overflow: ellipsis; — needs a width constraint.
Caret (typing cursor) The blinking vertical bar shown at the current insertion point in a text editor or terminal. @keyframes cursor-blink at 720 ms steps(2, end); project hook .home-terminal__caret.
Dotted underline (glossary mark) A dotted line under a term signaling “there is a definition on hover or focus”. The Pattern B tooltip uses this convention. text-decoration: underline dotted; with a paired tooltip.

Color & gradients (visual cheatsheet)

Color and gradient idioms used across surfaces, backgrounds, and theme tokens.

TermPlain-language definitionImplementation noteVisual example
Linear gradient Two or more colors blended along a straight line. Used for progress bars, premium buttons, hero accents. linear-gradient(165deg, #f3eeff, #ffffff)
Radial gradient Color that radiates out from a focal point. Used for vignettes, orbs, hero backgrounds. radial-gradient(circle at 30% 40%, color1, color2)
Conic gradient Color that sweeps around a center like a clock hand. Used for ring progress meters and donut charts. conic-gradient(from 0deg, #7c5cff, #22d3ee)
color-mix A CSS function that mixes two colors at a given ratio — used to derive hover, border, and shadow tints from theme tokens without hardcoding. color-mix(in srgb, var(--accent) 28%, var(--card))
Theme tokens (light/dark parity) Named CSS variables that change value when the theme switches. The same selector renders correctly in both themes by referencing the token, not the literal color. color: var(--text); background: var(--card); — set once, the theme decides the value.
Contrast pair A foreground/background combination tested against WCAG AA (≥4.5:1 for body, ≥3:1 for large text). Every theme token must pass. Audit with the contrast checker; failed pairs are flagged in the docs UI/UX assessment runs.

Motion patterns (visual cheatsheet)

Named motion idioms that show up in this codebase. Each row pairs the term with a hint of what it looks like in still SVG.

TermPlain-language definitionImplementation noteVisual example
Slide-in Element enters from off-screen by translating along one axis. Default for toasts (8 px from the bottom). transform: translateY(8px) → 0; over 200 ms.
Fade Opacity transition from 0 to 1 (or back). The cheapest, most reduced-motion-friendly transition. opacity: 0 → 1; 180–240 ms.
Scale (zoom) Element grows or shrinks. Subtle (0.98 → 1) for press feedback; large for lightbox open. transform: scale(0.98) → scale(1);
Stagger reveal A row of elements animates in one-by-one with a small delay so the eye reads them in order. Used for hero card sections. transition-delay: i * 65ms; set per child after a single rAF.
Parallax (depth) Background layers move slower than the foreground, creating an illusion of depth. The hero orbs use small parallax depth multipliers. Pointer offset × depth multiplier; depth values 0.12 / 0.18 / 0.28 in bindHeroParallax().
Shimmer sweep A diagonal translucent band sweeps across a card on hover, like light catching a surface. ::after overlay translateX(-110% → 110%) over 620 ms.
Pulse An element grows and fades repeatedly to draw attention without being aggressive — used by status dots and the rocket idle state. @keyframes pulse with scale 1 → 1.4 and opacity 1 → 0.
Scramble / decrypt Text first cycles through random characters and then "settles" into the real text, character by character — like a movie hacker reveal. startWordmarkScramble() / bindHeroDecrypt(); ~50 ms per character with per-char lock timestamps.
Magnetic CTA A button subtly slides toward the cursor when the cursor is near, as if attracted to it. data-home-magnetic hook → damped translate clamped to ±9/±7 px in bindMagneticCta().
Iris shutter A circular wipe that closes (or opens) like a camera aperture. Used to dismiss the home boot intro. clip-path: circle(150% → 0%) over 720 ms with steep cubic-bezier.
Lock-flash A brief brightness flash and scale pulse when a scrambling character settles to its final letter. CSS animation ig-lockflash (scale 1.18 → 0.98 → 1, 380 ms) on .ig-char--locked.
Scan beam A bright bar that sweeps once across an element, like a barcode scanner reading the wordmark. One-shot ig-beam animation on .home-intro__wordmark.is-locked.
Scanlines / vignette Faint horizontal stripes (CRT look) and a dark border-fade around the edges, used by the home boot intro to feel like an instrument display. .home-intro__scanlines 1 px-on / 3 px-off gradient + .home-intro__vignette radial gradient.

Accessibility patterns (visual cheatsheet)

The accessibility primitives the docs portal depends on. They are invisible by default — these visuals show what they do, not what they look like on screen.

TermPlain-language definitionImplementation noteVisual example
Skip link A keyboard-only link, hidden until Tab is pressed, that jumps the user past the navigation straight to the main content. injectDocsSkipToContentLink() in docs-nav.js; visible only on :focus-visible.
Focus trap Keyboard focus is constrained to the dialog and cannot Tab back to the page underneath. Required for modal dialogs. Quick-actions palette and diagram lightbox use aria-modal="true" + a Tab-cycle handler.
sr-only (visually hidden) Text rendered to screen readers but invisible on screen. Used for icon-only button labels and live-region messages. Project utility class clips the element to a 1×1 pixel and removes it from the layout while keeping it in the AT tree.
ARIA live region An element that announces new content to screen readers when it changes — without moving focus. Toasts use role="status" (polite); errors use role="alert" (assertive). aria-live="polite" on the toast root; one announcement per toast appended.
Landmark region A semantic container (<main>, <nav>, <aside>, <header>) that lets AT users jump between major page sections. Use the right tag, not <div role="…">. The internal layout has explicit landmarks for sidebar, main, and footer.
aria-current Marks the active item in a set — current page in a nav, current step in a wizard, current section in a TOC. aria-current="page" on the active link; AT reads "current page".
Roving tabindex In a list of related buttons (toolbar, menu, TOC), only one item is in the Tab order at a time. Arrow keys move within the list; Tab leaves it. One tabindex="0", others tabindex="-1"; arrow keys move focus and shift tabindex.

Performance & observers (visual cheatsheet)

Browser observers and timing primitives the docs runtime uses to stay smooth.

TermPlain-language definitionImplementation noteVisual example
IntersectionObserver A browser API that tells you when an element enters or leaves the viewport — without scroll listeners. Used by reveal-on-scroll, scroll spy for the in-page TOC, section tint shifting.
ResizeObserver Tells you when the size of an element changes — without window resize listeners. Used to relayout the in-page TOC when the sidebar drawer opens. Cheaper than recomputing on every animation frame; fires only on actual size change.
MutationObserver Tells you when DOM nodes are added, removed, or changed under a root. Used to re-attach behaviors to dynamically inserted content. Pair with a debounce so a burst of mutations is batched into one callback.
Throttle Run a handler at most once every N ms even if it's called many times. Used for scroll/resize listeners. Many calls in → at most one call per N ms out.
Debounce Wait until the burst of calls has stopped for N ms, then run the handler once. Used for the search input. Many calls in → exactly one call after the burst settles.
Layout shift (CLS) Visible content jumping after it has rendered — usually because a late-loading image or font changes the layout. Tracked by the Cumulative Layout Shift metric. Reserve space with aspect-ratio or fixed dimensions; preload critical fonts.
Reflow / repaint The browser recomputes geometry (reflow) or repaints pixels (repaint). Reading offsetTop right after writing styles forces a synchronous reflow — the most common perf bug in JS animations. Batch reads then writes; use rAF; avoid layout-trashing in scroll handlers.

Forms & input patterns (visual cheatsheet)

Named input and disclosure patterns. Use the term that matches the ARIA pattern — it tells reviewers exactly what keyboard model the widget owes them.

TermPlain-language definitionImplementation noteVisual example
Text input A single-line text field. The simplest form control; everything else inherits its keyboard expectations. <input type="text"> with a label or aria-label.
Select (native) A native dropdown chosen via the OS picker. Use it whenever the choices fit; native selects are accessible by default. <select> with <option> children. Skip the JS combobox dance if you can.
Combobox A text input + popup list. The user types to filter and arrow-keys to choose — like the docs portal command palette filter. ARIA role="combobox" + aria-controls + aria-activedescendant; popup is a listbox.
Listbox A scrollable list of options the user picks from with arrow keys. Used inside the combobox and the quick-actions palette. role="listbox" with role="option" children; manage aria-selected.
Checkbox A binary on/off toggle. Multiple may be checked at once; not mutually exclusive — that's a radio group. <input type="checkbox">; for ARIA versions role="checkbox" + aria-checked.
Radio group A set of options where exactly one is selected. The selected radio replaces the previously selected one. <input type="radio" name="g">; arrow keys move selection within the group.
Toggle (switch) A binary control that looks like a hardware toggle. Functionally equivalent to a checkbox; pick switch when the action is "turn on/off" rather than "include/exclude". role="switch" + aria-checked; transition the thumb position.
Accordion / disclosure A header that expands a hidden region when clicked. One disclosure = one panel; an accordion is a row of disclosures, often with mutual-exclusion. <details><summary> for the simple case; aria-expanded + aria-controls for the JS version.
Tabs A horizontal row of tab labels switching between mutually-exclusive panels. Useful when the panels are peers; not the right pattern for sequential steps. role="tablist" / role="tab" / role="tabpanel"; arrow keys move tab focus.
Dropdown menu A button that opens a small action list. Items run a command and close the menu — not selection persistence. role="menu" + role="menuitem"; Esc closes; arrow keys navigate.
Popover An anchored content panel — richer than a tooltip but lighter than a dialog. Stays open until dismissed; never traps focus. HTML popover attribute or scripted host with aria-haspopup="dialog".
Modal dialog A focused overlay panel that blocks the rest of the page. Used for the diagram lightbox and the quick-actions palette. role="dialog" + aria-modal="true" + a focus trap; restore focus to the launcher on close.
Drawer A panel that slides in from one edge of the viewport. The internal sidebar uses a left drawer on tablet/phone. role="dialog" + side anchor; close on backdrop click + Esc.
Sheet (bottom sheet) A modal-ish panel anchored to the bottom edge. Common on mobile when the content is too tall for an inline popover. Same dialog semantics as a drawer; anchor via bottom: 0 + slide-up animation.
Floating Action Button (FAB) A small persistent control pinned to a viewport corner with position: fixed; hovers above content and stays put while the page scrolls. One or two FABs per page max — never primary navigation. Examples in the docs portal: .docs-back-to-top (rocket) and .docs-report-bug-fab. Anchor via position: fixed + corner offset; respect env(safe-area-inset-*).

How to use this glossary

  • While reading code: if a CSS class or function name uses an unfamiliar word ("orb", "wordmark", "shimmer"), find it here first. The implementation note tells you exactly which selector or function to look at.
  • While writing PR descriptions: use these terms verbatim. "Adjusted radar pulse timing" beats "made the circle thing slower" — reviewers know exactly what changed.
  • While writing a new screen spec: use the same term across the spec, glossary, and code. Add new terms here in the same PR that introduces them in the screen.
  • If a term feels ambiguous: open an issue or fix it directly — glossary precision is the foundation of every other doc.

Usage guidelines

  • Use glossary terms in issue titles, task descriptions, and PR notes to reduce ambiguity.
  • When introducing a new frontend concept, add it here in the same change set as feature documentation updates.
  • When a term changes meaning, update this page and cross-references in related docs pages.
