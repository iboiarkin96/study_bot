Unified visual contract

Floating elements share one premium surface style via common CSS tokens:

  • --docs-overlay-border
  • --docs-overlay-bg
  • --docs-overlay-shadow
  • --docs-overlay-blur

Contract requirement: any new popup/overlay must use the same tokens unless there is a documented product reason to diverge.

Behavior contract

  • Toasts are non-blocking and must not trap focus.
  • Auto-dismiss toasts should have clear timing and reduced-motion-safe transitions.
  • Dialog-like overlays require semantic roles (role="dialog", aria-modal) and close affordances.
  • Desktop-only floating features (for example in-page TOC promo) must be disabled on tablet/phone where that UI is hidden.
  • Any action button in popup/toast must preserve existing page state and avoid destructive side effects.

Implementation entry points

  • docs/assets/docs.css — popup tokens and shared popup styling block.
  • docs/assets/docs-popups.js — centralized popup/toast runtime helpers (Sticky TOC + Continue reading prompts).
  • docs/assets/docs-nav.js — integration points, fallbacks, and page-level wiring.
  • docs/assets/docs-diagram-lightbox.js — diagram overlay behavior.
  • docs/assets/internal-sidebar.js — drawer overlay behavior for internal navigation.

QA checklist

  • All popup surfaces look visually consistent in light and dark themes.
  • No popup is clipped under sticky bars or under the internal drawer stack.
  • Desktop and tablet/phone policies are respected (including hidden desktop-only promos).
  • Keyboard navigation and focus-visible states are present and readable.

Do's and Don'ts

DoDon't
Reuse the unified surface tokens (--docs-premium-surface-*) when building a new floating surface. Re-invent the surface visuals for "just this one toast" — drift between popups is the most-flagged audit issue.
Add role="dialog" + aria-modal="true" + a focus trap for any modal overlay. Ship a modal that does not return focus to the launcher on close — keyboard and screen-reader users get stranded.
Auto-dismiss informational toasts after 3–5 s and animate respectfully under prefers-reduced-motion: reduce. Auto-dismiss error toasts — users miss them and lose data. Errors stay until acknowledged.
Hide desktop-only promos at tablet/phone widths via the documented breakpoint policy. Stack three popups on a 360 px screen — the screen becomes unusable.
