Runbook: In-page “On this page” sidebar missing or empty

Trigger

  • A long-form HTML page under docs/ (not docs/api/) shows no sticky “On this page” block, or the list is empty.
  • Expected: after the global doc nav, the main column and a right-hand (or stacked) TOC built by docs/assets/docs-nav.js.

Fast triage

  1. Confirm <script defer src="…/assets/docs-nav.js"></script> is present and loads (browser devtools → Network → no 404 for docs-nav.js).
  2. Confirm JavaScript is allowed (not disabled; for file:// some browsers restrict scripts — prefer a local static server, e.g. cd docs && python -m http.server 8765).
  3. Open devtools → Console: look for errors from docs-nav.js before layout runs.
  4. In Elements, check that <div id="docs-top-nav"> still exists after load (it must wrap the top nav — the script must not replace this node; it should contain <nav class="top-nav">).
  5. Confirm the page ends <main> with <div class="docs-inpage-toc-mount" data-inpage-toc="auto"></div> (last child of <main>).

Most common causes

  • Mount missing or wrong place — without the mount div, the TOC layout is not inserted.
  • No qualifying headings — the TOC is built from h2/h3 with id (assigned automatically where missing). p.lead is intentionally not listed.
  • Missing id on a heading — the script assigns stable ids from heading text when absent; if a heading still does not appear, check for invalid markup around it.
  • Diagram-only headings skipped — h3 inside figure.sys-diagram / .sys-diagram are excluded from the TOC.
  • Very long TOC lists — pages with many headings (for example the backlog index) still use the same mount; the link list scrolls inside the sidebar. If you truly want no sidebar, remove the mount from that page only.

Recovery steps

  1. Add the mount as the last node inside <main> (see Documentation pipeline → In-page “On this page” sidebar).
  2. Ensure each section you want in the TOC has an h2 or h3 with a stable id (or let the script generate one from heading text).
  3. Use clear h2/h3 structure; cards and sections are optional and do not affect which headings are listed.
  4. Run make docs-fix so HTML normalization stays consistent, then make docs-check.

Exit criteria

  • “On this page” appears with the expected section links; scroll-spy highlights the active link while scrolling.
  • No console errors from docs-nav.js on that page.
