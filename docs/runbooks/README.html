Operational Rule

  • If root cause reveals architecture/policy gap, capture decision in ADR.
  • If root cause is procedural, update the related runbook.

Mini-SLA

  • During active incident: add temporary notes/checklist in the relevant runbook within 30 minutes.
  • After incident resolution: publish finalized runbook update within 24 hours.
  • If same incident repeats twice in 30 days: update runbook and add preventive action item in developer docs/ADR.
