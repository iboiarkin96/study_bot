Overview

RFCs spell out how we build and run things: implementation detail, design rationale, and operational steps. ADRs record what we decided and why. An RFC usually follows an ADR and provides the specification the ADR intentionally omits. New RFCs start from the RFC template and follow the same lifecycle as ADRs (ADR 0018).

Client-side full-text search across all documentation pages. The program covers the index build pipeline, query-time ranking, telemetry, and SLO targets. The architecture decision is in ADR 0027.
